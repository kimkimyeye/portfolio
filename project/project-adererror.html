Problem Solving

GSAP scrolltrigger의 속도와 타이밍이 왜 이렇게 빠를까?

이번 프로젝트에서는 gsap 플러그인을 처음 사용해 보았는데,

로고가 커지면서 화면 전체를 덮어 배경 이미지가 되는 애니메이션을 계획했습니다.

로고의 scale 값이 100배 정도로 커져야 로고 전체가 화면을 덮을 수 있었고

큰 값으로 인해 스크롤을 조금만 돌려도 애니메이션이 너무 빠르게 진행되었습니다.

이를 해결하기 위해 GSAP 플러그인을 공부하여 start와 end 위치를 맞추고 

duration과 scrub 값을 적절하게 조절해 해결했습니다.

특정 구간에서만 가로 스크롤이 생길 때

AOS 플러그인을 사용하여 스크롤을 내릴 때 사진과 글자가 fade in 되는 이벤트를 구성했는데,

뷰포트 바깥에 있다가 안쪽으로 들어오면서 나타나는 효과가 적용되어서

스크롤이 해당 구역을 지날 때 가로 스크롤이 생기는 문제가 발생했습니다.

특정 섹션에 overflow: hidden; 속성을 추가해서 해결했습니다.

Review

팀 프로젝트로 3명의 팀원들과 함께 웹 디자인과 퍼블리싱을 진행했습니다.

인터렉션 요소를 많이 포함하여 모션이 화려한 홈페이지를 계획했는데

페이지 구성을 결정할 때 여러 가지 아이디어가 많이 나와 목표에 맞는 홈페이지를 제작할 수 있었습니다.

GSAP 스크롤 트리거를 처음 사용하여 모션을 구현했는데, start와 end 값을 지정하고

그 안에서 스크롤 위치에 따라 다양한 애니메이션을 구현할 수 있다는 점이 재미있었습니다.

이 기능을 통해 가로 스크롤 레이아웃이나 고정 화면에서 나타나는 효과 등

다양한 레이아웃을 구현할 수 있어서 좋은 기회였습니다.

팀 프로젝트에서 퍼블리싱 가이드라인을 제작 후 진행하였는데 퍼블리싱 가이드라인을 맞추니

코드 수정이 쉽고, 공동 작업이 수월해진 점을 알게 되었습니다.
